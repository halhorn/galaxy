# Story 1: 基本シミュレーション

## 目的

プロジェクトの基盤を構築し、1000体の物体がニュートン重力で引き合い動く様子を 3D で観察できる最小の MVP を完成させる。力の式は trait ベースで差し替え可能な設計にする。

## 計画概要

### Phase 1: プロジェクトセットアップ
Cargo プロジェクト作成、Bevy 導入、空のウィンドウ表示、開発環境整備 (clippy, fmt, fast compile 設定)。

### Phase 2: 物理エンジンコア
ECS コンポーネント設計 (Position, Velocity, Mass)、Force trait 定義、ニュートン重力実装、Velocity Verlet 積分器、O(N²) ブルートフォース計算。ヘッドレスでテスト可能にする。

### Phase 3: 3D レンダリング・統合
物体のインスタンスメッシュ描画、基本オービットカメラ、初期シナリオ設定、統合テスト。

## 設計メモ

- **Velocity Verlet**: シンプレクティック積分でエネルギーが長時間安定する。N体シミュレーションの標準的選択。
- **ソフトニング**: 距離 r → 0 での発散を防ぐため、力の計算で ε (softening length) を導入する。
- **Force trait**: `fn calculate(&self, bodies: &[Body]) -> Vec<Vec3>` のような形式で、ペアワイズ力に限らず任意の力を表現可能にする。

### 決定事項

- **単位系**: 天文単位 (AU, M☉, yr)。G = 4π² AU³/(M☉·yr²)。
- **初期シナリオ**: ディスク状回転分布 (銀河風)。円盤状に配置し、ケプラー回転速度を初速として与える。

## 受け入れ条件

- [ ] `cargo run` で 1000体の重力シミュレーションが 3D で表示される
- [ ] 60fps 以上で動作する
- [ ] マウスドラッグでオービット回転ができる（最小限のカメラ操作）
- [ ] Force trait を差し替えてニュートン重力以外の力も動かせる
- [ ] 物理テスト: 2体問題で楕円軌道が安定する（エネルギー保存を検証）
